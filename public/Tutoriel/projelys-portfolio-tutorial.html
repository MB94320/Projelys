Portefeuille projets — Vue d'ensemble

Project portfolio — Overview

La page Portefeuille Projets (/projects) est la vue centrale de pilotage multi-projets. Elle agrège et filtre dynamiquement l'ensemble du portefeuille en temps réel, sans rechargement serveur.

The Project Portfolio page (/projects) is the central multi-project steering view. It dynamically aggregates and filters the entire portfolio in real time, without server reload.

🔗 URL & données

🔗 URL & data

/projects
Données : props initialProjects (rendu serveur) Data: initialProjects props (server render)

Toutes les données sont chargées en une seule fois côté serveur et passées au composant client. Les filtres et KPIs se recalculent côté navigateur sans aller-retour API.

All data is loaded once server-side and passed to the client component. Filters and KPIs recalculate browser-side without API round trips.

Alert bell — Imminent deadlines

La cloche 🔔 en haut de la page affiche un badge rouge avec le nombre d'alertes non lues. Elle signale les projets dont l'échéance est dans les 7 prochains jours.

The 🔔 bell at the top of the page shows a red badge with the number of unread alerts. It signals projects with a deadline within the next 7 days.

📐 Logique de génération des alertes

📐 Alert generation logic

Pour chaque projet avec échéance : For each project with a deadline:
diffDays = (deadline – now) / ms_par_jour
Si 0 ≤ diffDays ≤ 7 → alerte "type: deadline" If 0 ≤ diffDays ≤ 7 → alert "type: deadline"
Message : "Projet [N°] : échéance dans N jours" Message: "Project [#]: deadline in N days"

📋 Panneau alertes

📋 Alerts panel

Cliquer la cloche ouvre un panneau flottant avec max. 10 alertes, triées par ordre décroissant de l'ID. Le badge est réinitialisé à 0 dès l'ouverture (notificationsRead = true).

Clicking the bell opens a floating panel with max. 10 alerts, sorted by descending ID. The badge resets to 0 on open (notificationsRead = true).

💡 Les alertes sont issues des projets filtrés

💡 Alerts come from filtered projects

Les alertes sont calculées depuis la liste allItems (tous les projets avec échéance, après filtres). Si vous filtrez par chef de projet, seules les alertes de ses projets apparaissent.

Alerts are calculated from the allItems list (all projects with a deadline, after filters). If you filter by PM, only their project alerts appear.

Logique de la colonne "Retard"

"Delay" column logic

La colonne Retard calcule dynamiquement si un projet est en retard, en comparant la date d'échéance à aujourd'hui et au statut du projet.

The Delay column dynamically calculates whether a project is late, comparing the deadline to today and the project status.

📐 Fonction getDelayLabel

📐 getDelayLabel function

Si pas de estimatedDate → null (pas de badge) If no estimatedDate → null (no badge)
Si statut ≠ "Terminé" ET estimated < now → "En retard" If status ≠ "Done" AND estimated < now → "Late"
Si statut = "Terminé" ET endDate > estimated → "Terminé en retard" If status = "Done" AND endDate > estimated → "Finished late"
Sinon → null (pas de badge) Otherwise → null (no badge)

🎨 Rendu du badge

🎨 Badge rendering

En retardLateBadge rose — projet non terminé dépassant son échéanceRose badge — unfinished project past its deadline
Terminé en retardFinished lateBadge rose — terminé mais après la date contractuelleRose badge — finished but after the contractual date
—Pas de badge si aucun retard détecté ou pas de date estiméeNo badge if no delay detected or no estimated date

💡 "En retard" vs "Terminé en retard"

💡 "Late" vs "Finished late"

La distinction est importante pour le reporting :

The distinction matters for reporting:

!
En retard = projet en cours (ou planifié) dont la date d'échéance est dépassée. Action corrective requise.
Late = in-progress (or planned) project whose deadline is past. Corrective action required.
~
Terminé en retard = projet terminé mais dont la date de fin réelle (endDate) est postérieure à l'échéance (estimatedDate). Retard historique, pour l'analyse.
Finished late = project completed but with an actual end date (endDate) later than the deadline (estimatedDate). Historical delay, for analysis.

Bonnes pratiques — Portefeuille

Best practices — Portfolio

🗓 Revue hebdomadaireWeekly review

Filtrer sur statut "En cours" chaque semaine. Vérifier les badges "En retard" et les KPIs avancement/risques. Ouvrir les projets à risque élevé via le bouton "MaJ projet".

Filter on status "In progress" each week. Check "Late" badges and progress/risk KPIs. Open high-risk projects via the "Update project" button.

🔔 Activer les alertesCheck alerts

Vérifier la cloche en début de semaine : elle signale les projets avec échéance dans 7 jours. Programmer des points de suivi spécifiques pour ces projets.

Check the bell at the start of each week: it signals projects with a deadline within 7 days. Schedule specific follow-up meetings for these projects.

📊 Utiliser les filtres chefUse PM filters

Le filtre "Chef de projet" permet à chaque manager de n'afficher que son portefeuille. Idéal pour les revues individuelles ou les exports personnalisés.

The "Project manager" filter lets each manager display only their portfolio. Ideal for individual reviews or personalized exports.

📅 Alimenter la timelineFeed the timeline

La timeline n'affiche que les projets avec une date d'échéance (estimatedDate). Renseigner cette date dans tous les projets actifs pour avoir une vue calendaire complète.

The timeline only shows projects with a deadline (estimatedDate). Fill this date in all active projects to get a complete calendar view.

📈 Renseigner les chargesFill in workloads

Le KPI "Charge prévue / consommée" n'est pertinent que si plannedLoadDays et consumedLoadDays sont à jour dans chaque tâche. Mettre à jour hebdomadairement.

The "Planned / consumed workload" KPI is only meaningful if plannedLoadDays and consumedLoadDays are up to date in each task. Update weekly.
