Contexte & navigation qualité

Context & quality navigation

La page Audits Projets est la vue qualité ISO 9001 centralisée de tous les audits du portefeuille. Elle mesure le niveau de conformité des processus projet par rapport à un référentiel structuré en 21 thèmes répartis en 2 domaines : AVV et Delivery.

The Project Audits page is the centralized ISO 9001 quality view of all portfolio audits. It measures project process conformity against a reference framework structured in 21 themes split across 2 domains: AVV and Delivery.

📐 Calcul taux précédent & actuel

📐 Previous & current rate calculation

Pour chaque projet : audit le plus récent = "actuel" For each project: most recent audit = "current"
Taux actuel = moy(globalConformityRate) des derniers audits Current rate = avg(globalConformityRate) of latest audits
Taux précédent = moy(previousGlobalRate) des derniers audits Previous rate = avg(previousGlobalRate) of latest audits

⚡ La différence entre les deux taux mesure la progression de la conformité entre la dernière et l'avant-dernière évaluation.

⚡ The difference between both rates measures progress in conformity between the latest and previous evaluation.

✅ Points forts — logique
✅ Strengths — logic
  • Taux actuel ≥ 85% → "Niveau de conformité élevé"
  • Current rate ≥ 85% → "High conformity level"
  • Taux 70–84% → "Conformité correcte, marges d'amélioration"
  • Rate 70–84% → "Good conformity, room for improvement"
  • Délai ≤ 15j → "Actions clôturées dans des délais raisonnables"
  • Delay ≤ 15d → "Actions closed within reasonable timeframes"
  • NC = 0 → "Aucune NC issue des audits"
  • NCs = 0 → "No NCs from audits"
⚠️ Points de vigilance — logique
⚠️ Watch points — logic
  • Taux < 70% → "Conformité insuffisante"
  • Rate < 70% → "Insufficient conformity"
  • Délai 15–30j → "Délai modéré"
  • Delay 15–30d → "Moderate delay"
  • Délai > 30j → "Délai élevé, risque de dérive"
  • Delay > 30d → "High delay, drift risk"
  • NC > 0 → "NC régulièrement émises"
  • NCs > 0 → "NCs regularly issued"

🔍 Données affichées

The arrow displays themes from the latest audit within the filtered scope (audit with the most recent evaluation date among filtered audits). Each smiley shows the theme label and rate % on hover.

💡 Lecture rapide

💡 Quick reading

Un maximum de smileys rouges sur les thèmes centraux (Planification, Performance, Risques) signale un projet en difficulté opérationnelle. Les thèmes en fin de flèche (Capitalisation, Insatisfactions) sont souvent les plus faibles.

Many red smileys on central themes (Planning, Performance, Risks) signals an operationally struggling project. End-of-arrow themes (Capitalisation, Dissatisfactions) are often the weakest.

Bonnes pratiques — Audits qualité

Best practices — Quality audits

📅 Cadence d'auditAudit cadence

Planifier un audit par projet tous les 3 à 6 mois. La comparaison précédent/actuel n'est pertinente que si les deux audits existent. Renseigner previousGlobalRate lors de chaque nouvel audit.

Schedule one audit per project every 3 to 6 months. The previous/current comparison is only meaningful if both audits exist. Fill previousGlobalRate with each new audit.

🎯 Cibler les thèmes rougesTarget red themes

Filtrer par taux <65% pour identifier les projets en difficulté. Cliquer "Audits" dans le tableau pour accéder à l'évaluation détaillée et identifier les thèmes <65% à prioriser.

Filter by rate <65% to identify struggling projects. Click "Audits" in the table to access the detailed evaluation and identify <65% themes to prioritize.

🔗 Lier NC et actionsLink NCs and actions

Toute NC émise lors d'un audit doit être liée à une action dans le plan d'actions projet. Vérifier que ncFromAuditCount est cohérent avec le registre NC.

Every NC raised during an audit must be linked to an action in the project action plan. Verify that ncFromAuditCount is consistent with the NC register.

📊 Suivre la progressionTrack progression

Utiliser le tableau projets pour identifier les projets dont le score actuel est inférieur au précédent (régression). Un filtre Chef de projet permet de faire la revue de portefeuille par manager.

Use the project table to identify projects where the current score is below the previous (regression). A PM filter enables portfolio review by manager.

⚙️ Configurer sous-traitance & X-shoreConfigure subcontracting & X-shore

Cocher "Oui" uniquement si ces modes sont réellement actifs sur le projet. Un thème activé mais non évalué affichera NA dans les graphiques et la flèche.

Check "Yes" only if these modes are actually active on the project. An activated but unevaluated theme will show NA in charts and the arrow.
