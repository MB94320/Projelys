Context & quality navigation

🎯 Définition d'une NC

🎯 NC definition

Une non-conformité est un écart entre ce qui est attendu (exigence) et ce qui est réalisé. Elle peut concerner un produit, un process, un document ou un service, et doit être traitée selon la démarche 8D ou équivalent.

A non-conformity is a gap between what is expected (requirement) and what is achieved. It can relate to a product, process, document or service, and must be treated using the 8D approach or equivalent.

Délai de clôture
Closure delay

Calcul du délai moyen de clôture

Average closure delay calculation

Le délai moyen de clôture mesure le temps moyen écoulé entre la date de détection et la date de clôture d'une NC. Il est calculé uniquement sur les NC disposant des deux dates.

The average closure delay measures the average time elapsed between detection date and closure date of a NC. It is calculated only on NCs with both dates.

📐 Formule

📐 Formula

Délai (j) = (closedDate – detectedOn) / ms_par_jour Delay (d) = (closedDate – detectedOn) / ms_per_day

Délai moyen = Σ(délais) / nb_NC_clôturées Avg delay = Σ(delays) / nb_closed_NCs

Si aucune NC n'est clôturée → délai moyen = 0. Les NC sans closedDate ou sans detectedOn sont exclues du calcul.

If no NC is closed → average delay = 0. NCs without closedDate or detectedOn are excluded from the calculation.

💡 Interprétation

💡 Interpretation

Un délai moyen élevé peut indiquer : manque de ressources sur les plans d'actions, NC sous-priorisées, causes racines non identifiées. Croiser avec le % NC critiques pour prioriser.

A high average delay may indicate: lack of resources on action plans, under-prioritized NCs, unidentified root causes. Cross with % critical NCs to prioritize.

Alertes & recommandations automatiques

Automatic alerts & recommendations

Projelys analyse les 3 KPIs clés et génère automatiquement un diagnostic en 3 blocs, mis à jour à chaque changement de filtre.

Projelys analyzes the 3 key KPIs and automatically generates a 3-block diagnosis, updated with each filter change.

✅ Points forts — logique
✅ Strengths — logic
  • NC ouvertes = 0 → "Aucune NC ouverte"
  • Open NCs = 0 → "No open NCs"
  • NC ouvertes ≤ 3 → "Volume maîtrisé"
  • Open NCs ≤ 3 → "Controlled volume"
  • % critiques ≤ 10% → "Faible proportion"
  • % critical ≤ 10% → "Low proportion"
  • Délai ≤ 5 j → "Clôture rapide"
  • Delay ≤ 5 d → "Fast closure"
⚠️ Points de vigilance — logique
⚠️ Watch points — logic
  • NC ouvertes > 3 → "Volume élevé"
  • Open NCs > 3 → "High volume"
  • % critiques 10–30% → "Part significative"
  • % critical 10–30% → "Significant share"
  • Délai 5–15 j → "Délai modéré"
  • Delay 5–15 d → "Moderate delay"
📝 Recommandations — logique
📝 Recommendations — logic
  • > 3 NC ouvertes → point de revue qualité
  • > 3 open NCs → quality review meeting
  • > 30% critiques → plan causes racines
  • > 30% critical → root cause plan
  • > 15 j délai → échéances 8D + rappels
  • > 15 d delay → 8D deadlines + reminders

🔴 Traiter les NC critiques en 1erTreat critical NCs first

Filtrer Sévérité:Critique + Statut:Ouvert pour isoler les NC urgentes. Chaque NC critique doit avoir un pilote 8D identifié et une échéance de traitement dans les 5 jours.

Filter Severity:Critical + Status:Open to isolate urgent NCs. Each critical NC must have an identified 8D pilot and a treatment deadline within 5 days.

📅 Renseigner closedDateFill closedDate

Sans closedDate, le délai moyen de clôture ne peut pas être calculé. Systématiser la saisie de cette date lors du passage à "Clôturé".

Without closedDate, the average closure delay cannot be calculated. Systematize entering this date when switching to "Closed".
